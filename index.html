1. Características Avanzadas de Python

Generadores y Expresiones Generadoras

Los generadores son funciones que producen una secuencia de resultados en lugar de calcular todo de una vez y devolver una sola vez. Son útiles para trabajar con secuencias grandes de datos sin usar mucha memoria.

# Definición de un generador de números Fibonacci
def fibonacci_generator():
    a, b = 0, 1
    while True:
        yield a
        a, b = b, a + b

# Uso del generador
fib = fibonacci_generator()
for _ in range(10):
    print(next(fib), end=' ')

# Expresión generadora para crear una secuencia de cuadrados
cuadrados = (x**2 for x in range(10))
print("\nCuadrados:", list(cuadrados))

Decoradores

Los decoradores son una forma de modificar o mejorar funciones o clases sin cambiar su código fuente. Son muy útiles para reutilizar código y aplicar patrones de diseño.

# Definición de un decorador
def mi_decorador(func):
    def wrapper():
        print("Algo está sucediendo antes de la función")
        func()
        print("Algo está sucediendo después de la función")
    return wrapper

# Uso del decorador
@mi_decorador
def saludo():
    print("¡Hola, mundo!")

saludo()

2. Programación Orientada a Objetos Avanzada

Herencia Múltiple y MRO (Method Resolution Order)

Python permite la herencia múltiple, donde una clase puede heredar de varias clases padre. El MRO define el orden en que se buscan los métodos en una jerarquía de clases.

class A:
    def quien_soy(self):
        print("Soy A")

class B(A):
    def quien_soy(self):
        print("Soy B")

class C(A):
    def quien_soy(self):
        print("Soy C")

class D(B, C):
    pass

d = D()
d.quien_soy()  # Imprime "Soy B" debido al MRO
print(D.mro())  # Muestra el orden de resolución de métodos

Métodos Mágicos (Dunder Methods)

Los métodos mágicos, también conocidos como métodos dunder (double underscore), permiten definir comportamientos especiales para las clases, como la representación de objetos y la sobrecarga de operadores.

class Libro:
    def __init__(self, titulo, autor, paginas):
        self.titulo = titulo
        self.autor = autor
        self.paginas = paginas

    def __str__(self):
        return f"{self.titulo} por {self.autor}"

    def __len__(self):
        return self.paginas

    def __del__(self):
        print(f"El libro {self.titulo} ha sido eliminado")

libro = Libro("Python Avanzado", "Sebastian Becerra", 300)
print(str(libro))
print(len(libro))
del libro

3. Manejo Avanzado de Datos

Comprehensions

Las comprehensions son una forma concisa de crear listas, diccionarios y conjuntos en Python, y permiten escribir código más limpio y eficiente.

# List comprehension
cuadrados = [x**2 for x in range(10)]
print("Cuadrados:", cuadrados)

# Dict comprehension
cuadrados_dict = {x: x**2 for x in range(5)}
print("Diccionario de cuadrados:", cuadrados_dict)

# Set comprehension
vocales_set = {letra for letra in "murcielago"}
print("Set de vocales:", vocales_set)

# Comprehension con condición
pares = [x for x in range(20) if x % 2 == 0]
print("Números pares:", pares)

Módulo Collections

El módulo collections proporciona alternativas especializadas a los contenedores integrados de Python, como defaultdict, Counter y namedtuple.

from collections import Counter, defaultdict, namedtuple

# Counter
texto = "mississippi"
print(Counter(texto))

# defaultdict
d = defaultdict(list)
d['frutas'].append('manzana')
d['frutas'].append('banana')
print(dict(d))

# namedtuple
Punto = namedtuple('Punto', ['x', 'y'])
p = Punto(11, y=22)
print(p.x, p.y)

4. Concurrencia y Paralelismo

Threading y Multiprocessing

Python ofrece módulos para trabajar con hilos y procesos múltiples, permitiendo la ejecución concurrente y paralela de código. Esto es útil para tareas que requieren mucho tiempo de CPU o I/O.

import threading
import multiprocessing
import time

def worker(num):
    print(f'Trabajador {num}')
    time.sleep(1)
    print(f'Trabajador {num} terminado')

# Threading
threads = []
for i in range(5):
    t = threading.Thread(target=worker, args=(i,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

# Multiprocessing
if __name__ == '__main__':
    processes = []
    for i in range(5):
        p = multiprocessing.Process(target=worker, args=(i,))
        processes.append(p)
        p.start()

    for p in processes:
        p.join()

Asyncio

Asyncio es un módulo para escribir código concurrente usando la sintaxis async/await. Permite manejar múltiples tareas concurrentes de manera eficiente.

import asyncio

async def contar():
    print("Uno")
    await asyncio.sleep(1)
    print("Dos")

async def main():
    await asyncio.gather(contar(), contar(), contar())

if __name__ == "__main__":
    import time
    s = time.perf_counter()
    asyncio.run(main())
    elapsed = time.perf_counter() - s
    print(f"{__file__} ejecutado en {elapsed:0.2f} segundos.")

5. Metaprogramación

Metaclases

Las metaclases son clases de clases, permitiendo modificar el comportamiento de la creación de clases. Son una herramienta poderosa para la creación de frameworks y bibliotecas.

class MiMetaclase(type):
    def __new__(cls, name, bases, attrs):
        # Convertir todos los métodos a mayúsculas
        uppercase_attrs = {
            key.upper(): value 
            for key, value in attrs.items() 
            if not key.startswith('__')
        }
        return super().__new__(cls, name, bases, uppercase_attrs)

class MiClase(metaclass=MiMetaclase):
    def metodo(self):
        return "Este es un método"

obj = MiClase()
print(obj.METODO())  # Funciona porque el método se convirtió a mayúsculas

Descriptores

Los descriptores definen cómo se accede a los atributos de una clase. Permiten personalizar el acceso, modificación y eliminación de atributos.

class Verbose_attribute():
    def __get__(self, obj, type=None) -> object:
        print("Accediendo al atributo")
        return 42
    
    def __set__(self, obj, value) -> None:
        print("Cambiando el atributo")
        
class Foo():
    atributo = Verbose_attribute()
        
f = Foo()
x = f.atributo
f.atributo = 21
